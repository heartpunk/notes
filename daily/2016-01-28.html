Links

  • Kyle Littler shared this cool blog post on how reducers are fuzzers, which he described as being about how the process of finding the minimal input that reproduces a bug can sometimes end up finding an instance of another bug. The example is a bit off from what's normally considered a reducer, and kind of weakens the piece rhetorically. For example, he's reducing the token sequence of a valid c++ program, rather than reducing the inputs to a typical program. True, he's trying to find bugs in the compiler, so I guess it's actually analogous, but it will certainly lose some people who might have otherwise understood the post.
